# Smart Tech Navigator — Video Demo Script
**Target length: 3–4 minutes | Format: Screen recording + voiceover**

---

## SETUP BEFORE RECORDING

- Open `index.html` in Chrome (full screen, zoom browser to 90%)
- Keep browser DevTools closed
- Use a clean browser profile (no extensions visible)
- Record screen at 1920×1080 if possible
- Speak at a calm, confident pace — pause 1 second after each key action

---

*Show: STN app loaded, welcome state, no input yet*

## [0:45 – 1:05] DEMO START — Load manufacturing scenario

*ACTION: Click the first scenario button — "Mid-size manufacturing ERP"*
*Show: The text area fills with the scenario description. Fields populate.*

>

*Pause 1 second. Point to the populated form.*

> "The agent takes this. And does the rest."

## [1:05 – 1:15] TRIGGER — Click the run button

*ACTION: Click "Analyse & Recommend"*
*Show: Button changes to loading state. Agent trace bar appears at top of right panel.*

> "One click. Three agents start working."

---

## [1:15 – 1:50] AGENT TRACE — Walk through the live log

*Show: Agent reasoning log appearing line by line. Trace dots activating.*

> "Watch the top bar — this is the agent pipeline in real time.
>
> First, the Intake Agent fires.
> It reads the business description and extracts structured requirements —

*Wait for the requirements card to appear below the log.*

> "And here — the structured output.
> Industry: Manufacturing. Size: mid-market.
> Required modules: supply chain, inventory, HR, payroll, GST integration.

## RECORDING TIPS

- **Go slow on the agent log** — let viewers read the lines appearing. This is the "magic moment."
- **Pause 2 seconds** before and after clicking "Analyse & Recommend" — build the anticipation.
- **Point with your cursor** to the score bars as you describe each dimension.
- **Don't rush the Gantt** — it's visually impressive and rarely seen in AI demos.
- If you make a mistake, keep recording — edit in post rather than restarting from scratch.
- Record the voiceover separately if your mic quality is better that way; sync in post.
